RoadGuard ML - חיזוי מדד מצב המיסעה (PCI)
הבעיה שפותרים
ניהול תחזוקת כבישים מתבצע כיום לרוב בצורה ידנית או בלוחות זמנים קבועים, מה שמוביל לבזבוז משאבים או להזנחה של תשתיות מסוכנות. הבעיה היא חוסר ביכולת לחזות מתי איכות האספלט תרד מתחת לסף הבטיחותי. פתרון מבוסס נתונים יאפשר אופטימיזציה של תקציבי סלילה ותחזוקה מונעת מצילת חיים.

קהל יעד
מהנדסי ערים במחלקות תשתיות ומנהלי פרויקטים בחברות תשתית (כמו נתיבי ישראל) הזקוקים לכלי תומך החלטה לתעדוף עבודות סלילה.

מקור הנתונים
נשתמש במאגר הנתונים הפתוח של עיריית שיקגו בנושא מצב כבישים (Road Surface Conditions).
קישור ישיר : https://www.kaggle.com/datasets/probelm/pavement-condition-index

ארכיטקטורה טכנית
מודל ML: רגרסיה מסוג Random Forest לחיזוי ציון ה-PCI (0-100).

ספריות עיקריות: pandas, scikit-learn, plotly, streamlit.

Input: גיל הכביש, עומס תנועה יומי, עובי שכבת אספלט ופרמטרי מזג אוויר.

Output: חיזוי ציון מצב הכביש והמלצה אופרטיבית (תחזוקה/סלילה/שיקום).

3 User Stories
כמתכנן ערים, אני רוצה להזין עומס תנועה צפוי כדי לקבל הערכה מתי הכביש יזדקק לקרצוף וסלילה מחדש.

כמנהל תקציב, אני רוצה לראות מפה עם סימון כבישים בסיכון גבוה כדי להחליט לאן להפנות את תקציב השנה הבאה.

כמהנדס ביצוע, אני רוצה להשוות בין סוגי תערובות אספלט שונות ולראות איך הן משפיעות על אורך חיי המיסעה.
